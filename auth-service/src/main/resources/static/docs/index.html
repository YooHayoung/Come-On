로그인

로그인의 경우, 다음 경로로 요청합니다.
지원하는 providerName을 경로 파라미터로 지정하고, 로그인 성공시, 리다이렉트 할 URL을 지정하여 요청하면, providerName에 맞는 소셜 로그인 화면을 응답으로 내리고, 사용자가 로그인을 진행할 수 있도록 합니다.

GET /oauth2/authorize/{providerName}?redirect_uri={redirect_url}

Table 5. /oauth2/authorize/{providerName}
파라미터 이름 설명

providerName

소셜 로그인 서비스 제공 벤더 코드
Table 6. 쿼리 파라미터
파라미터 이름 필수값 여부 설명

redirect_uri

true

로그인 성공시, 토큰을 전달받을, 프론트측 리다이렉트 URL

정상 흐름 : 로그인에 성공한 경우

소셜 로그인을 성공적으로 진행하고, 애플리케이션 서버에서의 로그인 과정이 성공적으로 진행 완료되면, 지정한 redirect_url로 이동하고, redirect_url의 쿼리 파라미터로 token과 expiry, userId를 담아줍니다.

token은 발급받은 AccessToken 값이며, 이는 expiry까지 유효합니다.
expiry는 UNIX TIME 형태를 가지는 숫자값 입니다.
userId는 유저를 식별하기 위한 숫자값입니다.

AccessToken은 인증이 필요한 요청마다 필수로 전송되어야 합니다.

또한 로그인에 성공하면, HttpOnly, Secure 쿠키로 RefreshToken을 담아서 반환합니다.
RefreshToken은 AccessToken이 만료되면, AccessToken을 재발급 받기위한 토큰으로, 30일간 유효합니다. RefreshToken은 로그인 요청시 지정한 redirect_uri의 도메인을 쿠키의 도메인으로 지정합니다.

예외 흐름 : 로그인에 실패한 경우

소셜 로그인에는 성공했으나, 내부 오류 등 여러 이유로 인하여 로그인에 실패하게 되면, 지정한 redirect_url로 이동하고, 쿼리 파라미터로 error를 담아서 반환합니다.

error 파라미터는 오류에 대한 메시지로 문자열 형태입니다.

로그아웃

  • 소셜 계정 로그아웃 : POST(GET) /oauth2/logout

  • 서버 로그아웃 : POST(GET) /auth/logout

소셜 계정 로그아웃

  • POST(GET) /oauth2/logout
