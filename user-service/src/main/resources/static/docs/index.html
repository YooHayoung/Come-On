공통 스펙

공통 응답 스펙

Table 1. 공통 응답 스펙
이름 설명

responseTime

응답 시간을 반환합니다.

code

응답 코드를 반환합니다.

data

결과 데이터를 반환합니다.

User-Service 응답메시지는 모두 위와 같은 스펙으로 구성되어 있습니다.

code field : 응답 코드 반환값
Table 2. 응답 코드
이름 설명

BAD_PARAMETER

요청 파라미터가 잘못되었습니다.

FORBIDDEN

권한이 없습니다.

NOT_FOUND

리소스를 찾지 못했습니다.

SERVER_ERROR

서버 에러입니다.

SUCCESS

요청이 성공하였습니다.

UNAUTHORIZED

인증에 실패하였습니다.

공통 응답 스펙의 code 필드에는 위와 같은 값들이 반환될 수 있습니다.
응답 코드는 SUCCESS인 경우를 제외하고 모두 예외가 발생한 상황입니다.

공통 응답 스펙 - 예외인 경우

Table 3. 예외 응답 스펙
이름 설명

code

API 내부에서 지정한 예외 코드를 반환합니다.

message

예외 메시지를 반환합니다.

예외가 발생한 경우에는 data 필드에 위와 같은 형식으로 발생한 예외에 대한 응답이 들어갑니다.
아래의 예시를 참고해주세요.

{
  "responseTime" : "2022-09-04T00:58:24.747892",
  "code" : "SERVER_ERROR",
  "data" : {
    "code" : 900,
    "message" : "죄송합니다. 서버 내부 오류입니다."
  }
}
errorCode field : 예외 응답 코드 반환값
Table 4. 예외 응답 코드
이름 설명

900

죄송합니다. 서버 내부 오류입니다.

901

파일이 비어있습니다. 확인해주세요.

902

파일 업로드에 실패하였습니다.

903

요청 데이터 검증에 실패하였습니다.

904

해당 식별자를 가진 리소스가 없습니다.

905

요청을 수행할 권한이 없습니다.

906

탈퇴 처리된 회원입니다.

907

인증된 사용자만이 이용 가능합니다.

예외 응답 스펙의 code 필드에는 위와 같은 값들이 반환될 수 있습니다.

1. 회원 정보 저장

POST /users

1) 정상 흐름

요청 예시

POST /users HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 142
Host: localhost:8080

{
  "oauthId" : "12345",
  "provider" : "KAKAO",
  "name" : "testName1",
  "email" : "email1@email.com",
  "profileImgUrl" : "profileImgUrl"
}

2. 유저 리스트 조회

여러 유저 리스트를 조회하기 위해 다음 경로로 요청합니다.

GET /users

1) 정상 흐름

요청 경로의 쿼리 파라미터로 조회할 유저들의 식별자를 입력하여 요청합니다.

요청 예시

GET /users?userIds=1,2,3,4,5 HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080

요청 파라미터

Table 9. 요청 파라미터
파라미터 이름 필수값 여부 설명

userIds

true

조회할 유저 식별값 리스트

요청을 성공적으로 처리하면 다음과 같은 응답을 반환합니다.
탈퇴한 회원의 경우, nickname, profileImgUrl 필드가 존재하지 않습니다.

응답 예시

HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 697

{
  "responseTime" : "2022-09-04T00:58:48.963772",
  "code" : "SUCCESS",
  "data" : {
    "count" : 5,
    "contents" : [ {
      "userId" : 1,
      "nickname" : "userName1",
      "profileImgUrl" : "http://file-domain/storedFileName1",
      "status" : "ACTIVATE"
    }, {
      "userId" : 2,
      "status" : "WITHDRAWN"
    }, {
      "userId" : 3,
      "nickname" : "userName3",
      "profileImgUrl" : "http://file-domain/storedFileName3",
      "status" : "ACTIVATE"
    }, {
      "userId" : 4,
      "status" : "WITHDRAWN"
    }, {
      "userId" : 5,
      "nickname" : "userName5",
      "profileImgUrl" : "http://file-domain/storedFileName5",
      "status" : "ACTIVATE"
    } ]
  }
}

응답 필드

Table 10. 응답 필드
필드명 타입 설명

Number

조회한 유저의 수

contents[].status

String

유저의 상태
