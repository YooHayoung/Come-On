0.0.2 → 0.0.3 변경 사항

  1. 모든 API에서 요청 헤더, 경로변수 정보도 신규로 문서화 했습니다.
    API에서 응답되는 더미 데이터의 값들이 이전 버전과 다를 수 있습니다.

  2. 모임 날짜, 모임 API의 예외 케이스가 추가되었습니다.

  3. 기타 오탈자를 수정했습니다.
    오탈자나 문맥 상 어색한 설명을 발견하실 경우 제보 부탁드립니다.

0.0.1 → 0.0.2 변경 사항

  1. 모임 리스트 조회 응답 데이터가 바뀌었습니다.

  3. 모임 코드 수정 API가 신규 등록되었습니다.
    요청 헤더, 경로변수 정보도 신규로 문서화 했습니다. (추후 모든 API 문서 수정 예정)

공통 스펙

공통 응답 스펙

이름 설명

responseTime

응답 시간을 반환합니다.

code

응답 코드를 반환합니다.

data

결과 데이터를 반환합니다.

모임 API의 응답메시지는 모두 위와 같은 스펙으로 구성되어 있습니다.

code 필드에 반환되는 값들

이름 설명

SUCCESS

요청이 성공하였습니다.

SERVER_ERROR

서버 에러입니다.

UNAUTHORIZED

인증에 실패하였습니다.

BAD_PARAMETER

요청 파라미터가 잘못되었습니다.

NOT_FOUND

리소스를 찾지 못했습니다.

FORBIDDEN

권한이 없습니다.

공통 응답 스펙의 code 필드에는 위와 같은 값들이 반환될 수 있습니다.
응답 코드는 SUCCESS인 경우를 제외하고 모두 예외가 발생한 상황입니다.

공통 응답 스펙 - 리스트인 경우

리소스의 리스트를 조회하는 경우에는 data 필드에 아래와 같은 스펙으로 리스트 정보를 응답합니다.

{
  "responseTime" : "2022-09-07T18:41:51.6405605",
  "code" : "SUCCESS",
  "data" : {
    "currentSlice" : 0,
    "sizePerSlice" : 5,
    "numberOfElements" : 3,
    "hasPrevious" : false,
    "hasNext" : false,
    "contents" : [ {
      "name" : "sample1",
      "id" : "1"
    }, {
      "name" : "sample2",
      "id" : "2"
    }, {
      "name" : "sample3",
      "id" : "3"
    } ],
    "first" : true,
    "last" : false
  }
}

여기서 Slice는 Page라고 생각하시면 됩니다.
모임 API는 Pagination이 아니라, Infinite Scroll로 리스트를 응답합니다.
따라서 totalCount 정보는 응답하지 않으며, hasNext가 true일 경우에 페이지 번호를 하나 증가시켜 서버로 요청을 보내주시면 다음 데이터를 응답해드립니다.
아래의 설명을 참고해주세요.

이름 설명

currentSlice

현재 슬라이스 번호를 표시합니다. 0부터 시작합니다.

sizePerSlice

한 슬라이스당 몇 개의 요소를 응답하는지 표시합니다.

numberOfElements

현재 슬라이스에 몇 개의 요소가 들어있는지 표시합니다.

hasPrevious

이전 슬라이스가 있는지 표시합니다.

hasNext

다음 슬라이스가 있는지 표시합니다.

contents

실질적으로 요청에 따라 응답되는 데이터가 들어갑니다.

first

처음 슬라이스인지 표시합니다.

last

마지막 슬라이스인지 표시합니다.

예외 코드 정리
아래는 모임 API에서 발생할 수 있는 예외 코드를 정리해 놓았습니다.

이름 설명

101

업로드 파일이 없는 경우 발생합니다.

102

파일 업로드에 실패할 경우 발생합니다. 다시 시도하더라도 오류가 발생한다면 문의 부탁드립니다.

103

요청 데이터 검증에 실패했을 경우 발생합니다.

104

해당 식별자를 가진 리소스가 없을 경우 발생합니다.

105

해당 모임에 유저가 속해있지 않을 경우 발생합니다.

106

서버 측 오류입니다. 추후에도 해결될 가능성이 없기에 오류 해결 문의 부탁드립니다.

107

해당 초대코드를 가진 모임이 없는 경우 발생합니다.

108

해당 초대코드가 만료된 경우 발생합니다.

109

이미 모임에 가입한 회원인 경우 발생합니다.

110

이미 해당 날짜를 선택한 회원인 경우 발생합니다.

111

서버 측 오류입니다. 추후에도 해결될 가능성이 없기에 오류 해결 문의 부탁드립니다.

112

기간(시작일과 종료일이 있는 날짜)이 잘못된 경우 발생합니다. 시작일보다 종료일이 이른 경우가 대표적입니다.

113

날짜가 기간(모임의 기간)내에 포함되지 않을 경우 발생합니다.

114

요청 데이터가 없거나, 데이터의 형식이 잘못되었을 경우에 발생합니다.

115

해당 회원이 해당 날짜를 선택하지 않았습니다.

116

해당 회원이 해당 모임의 주인이 아니기에 권한이 없을 경우 발생합니다.

117

코드를 갱신할 때, 해당 초대코드가 아직 만료되지 않았을 경우 발생합니다.

118

유저 권한을 수정할 때 HOST로 변경 시에 발생합니다. (HOST 변경은 막혀있습니다.)

119

HOST 권한을 가진 유저의 권한을 수정하려고 할 경우 발생합니다. (HOST는 권한 수정 불가능입니다.)

120

요청 회원이 해당 요청을 처리할 권한이 없을 경우 발생합니다.

121

서버 측 오류입니다. 추후에도 해결될 가능성이 없기에 오류 해결 문의 부탁드립니다.

2. 장소 수정 - /meetings/{meetingId}/places/{placeId}

모임 장소를 수정기능은 장소 정보(name, lat, lng), 장소 메모(memo), 장소 순서(order)를 수정할 수 있도록 설계되어 있습니다.

name, lat, lng 필드만 요청 데이터로 보낸다면, 장소 정보만 수정됩니다.
memo 필드만 요청 데이터로 보낸다면, 장소 메모만 수정됩니다.
order 필드만 요청 데이터로 보낸다면, 장소 순서만 수정됩니다.

추가로 세 가지 수정 기능을 한번에 이용할 수도 있습니다.
예를 들어 name, lat, lng, memo, order 필드를 모두 요청 데이터로 보낸다면, 장소 정보, 메모, 순서 모두 수정 가능합니다.

모임 날짜

1. 날짜 생성 - /meetings/{meetingId}/dates

모임과 날짜는 1:N 관계를 이루고 있습니다.
그리고 해당 날짜를 선택한 회원을 저장하기 위해 날짜유저라는 테이블이 있고, 이는 날짜와 1:N 관계를 이루고 있습니다.

따라서 해당 문서에서 얘기하는 날짜는 모임에서 선택된 날짜를 저장하기 위한 날짜 테이블의 데이터를 얘기하며, 날짜유저라는 말은 해당 날짜를 선택한 회원들의 데이터를 의미합니다.

1) 정상 흐름

날짜 저장의 경우에는 크게 2가지 경우로 나뉩니다.
1) 해당 날짜를 아무도 선택하지 않아서 날짜 데이터가 없는 경우
2) 해당 날짜를 이미 누군가가 선택하여 날짜 데이터가 있는 경우

두 경우 모두 날짜 저장 API를 이용하시면 됩니다. 만약 날짜 데이터가 없다면, 날짜 데이터를 새로 생성하고, 해당 날짜를 선택한 날짜유저 데이터도 새로 생성합니다.
