공통 스펙

공통 응답 스펙

이름 설명

responseTime

응답 시간을 반환합니다.

code

응답 코드를 반환합니다.

data

결과 데이터를 반환합니다.

모임 API의 응답메시지는 모두 위와 같은 스펙으로 구성되어 있습니다.

code 필드에 반환되는 값들

이름 설명

SUCCESS

요청이 성공하였습니다.

SERVER_ERROR

서버 에러입니다.

UNAUTHORIZED

인증에 실패하였습니다.

BAD_PARAMETER

요청 파라미터가 잘못되었습니다.

NOT_FOUND

리소스를 찾지 못했습니다.

공통 응답 스펙의 code 필드에는 위와 같은 값들이 반환될 수 있습니다.
응답 코드는 SUCCESS인 경우를 제외하고 모두 예외가 발생한 상황입니다.

공통 응답 스펙 - 리스트인 경우

리소스의 리스트를 조회하는 경우에는 data 필드에 아래와 같은 스펙으로 리스트 정보를 응답합니다.

{
  "responseTime" : "2022-08-19T12:19:06.1947953",
  "code" : "SUCCESS",
  "data" : {
    "currentSlice" : 0,
    "sizePerSlice" : 5,
    "numberOfElements" : 3,
    "hasPrevious" : false,
    "hasNext" : false,
    "contents" : [ {
      "name" : "sample1",
      "id" : "1"
    }, {
      "name" : "sample2",
      "id" : "2"
    }, {
      "name" : "sample3",
      "id" : "3"
    } ],
    "last" : false,
    "first" : true
  }
}

여기서 Slice는 Page라고 생각하시면 됩니다.
모임 API는 Pagination이 아니라, Infinite Scroll로 리스트를 응답합니다.
따라서 totalCount 정보는 응답하지 않으며, hasNext가 true일 경우에 페이지 번호를 하나 증가시켜 서버로 요청을 보내주시면 다음 데이터를 응답해드립니다.
아래의 설명을 참고해주세요.

이름 설명

currentSlice

현재 슬라이스 번호를 표시합니다. 0부터 시작합니다.

sizePerSlice

한 슬라이스당 몇 개의 요소를 응답하는지 표시합니다.

numberOfElements

현재 슬라이스에 몇 개의 요소가 들어있는지 표시합니다.

hasPrevious

이전 슬라이스가 있는지 표시합니다.

hasNext

다음 슬라이스가 있는지 표시합니다.

contents

실질적으로 요청에 따라 응답되는 데이터가 들어갑니다.

first

처음 슬라이스인지 표시합니다.

last

마지막 슬라이스인지 표시합니다.

공통 응답 스펙 - 예외인 경우

예외가 발생한 경우에는 data 필드에 아래와 같은 형식으로 발생한 예외에 대한 응답이 들어갑니다.

{
  "responseTime" : "2022-08-19T12:19:06.2278021",
  "code" : "BAD_PARAMETER",
  "data" : {
    "code" : 104,
    "message" : "Error Message"
  }
}

아래의 설명를 참고해주세요.

이름 설명

code

API 내부에서 지정한 예외 코드를 반환합니다.

message

예외 메시지를 반환합니다.

예외 코드 정리
아래는 모임 API에서 발생할 수 있는 예외 코드를 정리해 놓았습니다.

이름 설명

101

업로드 파일이 없는 경우 발생

102

파일 업로드에 실패할 경우 발생

103

요청 데이터 검증에 실패했을 경우 발생

104

해당 식별자를 가진 리소스가 없을 경우 발생

3. 모임 삭제 - /meetings/{meetingId}

모임 삭제 요청은 사실 회원의 모임 탈퇴 요청과 동일합니다.
모임과 회원은 N:M관계를 이루고 있기 때문에, 어느 한 회원이 모임을 삭제하게 되면 다른 회원도 의도치 않게 모임이 삭제되어버리기 때문입니다.
따라서 실제 모임 리소스가 삭제되는 경우는 더 이상 모임에 회원이 남아있지 않을 경우 단 한가지 입니다.

1) 정상 흐름

요청 예시

DELETE /meetings/10 HTTP/1.1
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOjEsIm5hbWUiOiJ0ZXN0IiwiaWF0IjoxNTE2MjM5MDIyfQ.0u81Gd1qz_yiMpa3WFfCQRKNdGx3OPiMCLm4ceBgbFw
Host: localhost:8080

삭제할 모임의 ID를 경로변수에 담아 DELETE 메서드로 호출해주세요.

응답 예시

HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 77

{
  "responseTime" : "2022-08-19T12:19:05.0532329",
  "code" : "SUCCESS"
}

응답코드가 SUCCESS로 응답된다면 정상적으로 처리된 것입니다. 응답 데이터는 따로 없습니다.

2) 예외 - 잘못된 경로변수를 보낼 경우

요청 예시

DELETE /meetings/5 HTTP/1.1
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOjEsIm5hbWUiOiJ0ZXN0IiwiaWF0IjoxNTE2MjM5MDIyfQ.0u81Gd1qz_yiMpa3WFfCQRKNdGx3OPiMCLm4ceBgbFw
Host: localhost:8080

위의 예시는 5번 식별자를 가진 모임이 없음에도 서버에 삭제 요청을 보낸 경우입니다.

응답 예시

HTTP/1.1 400 Bad Request
Content-Type: application/json
Content-Length: 191

{
  "responseTime" : "2022-08-19T12:19:05.0022222",
  "code" : "BAD_PARAMETER",
  "data" : {
    "code" : 104,
    "message" : "해당 ID와 일치하는 모임이 없습니다."
  }
}
